# Security Policy

## Supported Versions

| Version | Supported |
|----------|-----------|
| Development (main) | ✅ |
| Older versions | ❌ |

## Reporting a Vulnerability

If you discover a security vulnerability, please do not create a public GitHub issue.

Instead, contact the maintainer privately with:

- Description of the vulnerability
- Steps to reproduce
- Proof of concept (if available)
- Potential impact

Please allow reasonable time for investigation before public disclosure.

## Scope

Security issues include, but are not limited to:

- Kernel privilege escalation
- Memory corruption
- Buffer overflows
- Integer overflows
- Use-after-free
- Race conditions
- Filesystem corruption
- Driver vulnerabilities

## Response Process

The maintainer will:

1. Confirm the report.
2. Investigate the issue.
3. Develop a fix.
4. Publish the fix.
5. Credit the reporter (if desired).

Thank you for helping keep NasuaOS secure.
